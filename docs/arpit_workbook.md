## What You Need to Learn/Review

### 1. MCP (Model Context Protocol) Servers (Priority: 🔴 Critical)

You need to understand how MCP servers work because you're routing all agent traffic through them.

**Key concepts:**

- MCP servers expose "tools" via JSON-RPC 2.0
- Each tool has a name, description, and input schema
- Agents call tools through the MCP Gateway you build
- Transport: STDIO (local) or HTTP (remote)

**Resources:**

#### Step 2: Go API Server

```go
// cmd/api/main.go — skeleton
package main

import (
    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.Default()

    // API routes
    api := r.Group("/api/v1")
    {
        // Agent status
        api.GET("/agents/status", getAgentStatus)

        // Test management
        api.POST("/tests/trigger", triggerTestRun)
        api.GET("/tests/results/:session_id", getTestResults)
        api.GET("/tests/history", getTestHistory)

        // RCA reports
        api.GET("/rca/:id", getRCAReport)

        // PR management
        api.GET("/prs", listAutoGeneratedPRs)
        api.POST("/prs/:id/approve", approvePR)

        // Metrics
        api.GET("/metrics/summary", getMetricsSummary)
    }

    // WebSocket for real-time updates
    r.GET("/ws", handleWebSocket)

    r.Run(":8080")
}
```

```go
type WSMessage struct {
    Type    string      `json:"type"`    // "agent_status" | "test_output" | "notification"

// Agent status update
type AgentStatus struct {
    AgentName string `json:"agent_name"` // "architect" | "scripter" etc.
    Status    string `json:"status"`     // "idle" | "running" | "error"

// Live terminal line

    SessionID string `json:"session_id"`
    Line      string `json:"line"`

}
```

```
Agent → MCP Gateway → Route based on tool name → MCP Server → Result → Agent

Tool Routing Table:
┌────────────────────────┬─────────────────────────┐
```

## Key Decisions for You

1. **MCP Gateway language**: Go or Node.js — Go for consistency with API server, Node.js if you want easier JSON-RPC handling. Either works.
2. **Database**: PostgreSQL for everything (test results, RCA reports, agent logs, PR tracking)
3. **Start with Docker Compose** — don't touch K8s until Phase 4
4. **WebSocket library**: Use `gorilla/websocket` in Go
5. **Your API server is the single entry point** for the frontend — Himanshu's Next.js never talks directly to the AI engine or MCP servers
